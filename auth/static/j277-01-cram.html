1. Systems Architecture

  • CPU Components: Control Unit (CU), Arithmetic/Logic Unit (ALU), Registers
  • Fetch–Decode–Execute Cycle:
    1. Fetch: Program Counter → Memory Address Register; read into Memory Data Register.
    2. Decode: Control Unit interprets instruction.
    3. Execute: ALU performs operation; result → Accumulator or memory.
    4. Increment PC: Point to next instruction.
  • Performance Factors: Clock speed, Number of cores, Cache size (L1, L2, L3).
  • Embedded Systems: Dedicated controllers inside devices (e.g. traffic lights).

3. Networks & Protocols

3.1 Network Hardware
  • NIC (Network Interface Card): An internal hardware component that provides a unique MAC address, enabling a device to send and receive data frames over Ethernet or wireless networks.
  • Switch: A Layer 2 device that maintains a MAC address table to forward data frames only to the destination port, reducing network congestion.
  • Router: A Layer 3 device that examines IP addresses to route packets between different networks, often providing NAT and DHCP services.
  • Wireless Access Point (WAP): A device that connects to a wired network and transmits data over radio waves, allowing wireless devices to join the network.

6. Essay Tips & Model Response

Sample 8-mark Question (June 2024):

“Discuss the features, benefits and drawbacks of open-source vs proprietary software for a computer game, and make a recommendation.”

Model Response:

Introduction: Define open-source (source code available, community-driven) and proprietary (closed-source, vendor-controlled). State you will compare and recommend.

Point 1 – Features: Open-source allows code modification; proprietary offers official support.

  • Explain: Modifications enable custom features quickly.
  • Example: Minecraft modding community vs. Unity engine licences.
  • Link: Custom game engines suit indie developers.

Point 2 – Benefits: Open-source is low-cost; proprietary has guaranteed updates.

  • Explain: Budget-friendly vs. reliable vendor patches.
  • Example: Godot engine vs. Unreal Engine subscription.
  • Link: Indie vs. AAA studios’ needs.

Point 3 – Drawbacks: Open-source may lack documentation; proprietary can be expensive.

  • Explain: Community support varies; licensing fees add up.
  • Example: Sparse tutorials vs. Unity’s tiered pricing.
  • Link: Support requirements for project scope.

Recommendation: For small teams choose open-source for flexibility and cost; for large studios choose proprietary for professional support.

Conclusion: Balanced summary emphasising trade-offs and final recommendation based on project scale.
