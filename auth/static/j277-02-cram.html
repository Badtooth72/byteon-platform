1. Computational Thinking

Break a problem down, spot patterns, ignore irrelevant detail, then create a step-by-step solution.
  • Decomposition: split a big problem into smaller parts.
  • Pattern recognition: notice similarities to previous problems.
  • Abstraction: focus only on the important details.
  • Algorithmic thinking: design a precise sequence of steps.

2. Algorithms

An algorithm is a finite set of instructions used to solve a problem.

Common algorithm constructs
  • Sequence: instructions run in order.
  • Selection: decisions such as IF, ELSE, CASE.
  • Iteration: repetition such as FOR, WHILE, REPEAT UNTIL.
Searching and sorting you should know
  • Linear search: checks each item in turn.
  • Binary search: repeatedly halves a sorted list.
  • Bubble sort: swaps adjacent items until sorted.
  • Merge sort: split, sort, then merge back together.
Example linear search index = 0 found = False WHILE index < LENGTH(list) AND found == False IF list[index] == target THEN found = True ELSE index = index + 1 ENDIF ENDWHILE

3. Programming Fundamentals

Variables and constants
  • A variable stores a value that can change.
  • A constant stores a value that should not change.
Operators
  • Arithmetic: +, -, *, /, MOD, DIV
  • Relational: =, <>, <, >, <=, >=
  • Logical: AND, OR, NOT
Subprograms
  • Procedure: carries out a task.
  • Function: returns a value.
Benefits of subprograms
  • Reusable code
  • Easier testing
  • Simpler maintenance
  • Improved readability

6. Defensive Design, Testing and Errors

Defensive design
  • Input validation
  • Authentication
  • Range checks
  • Length checks
  • Presence checks
  • Type checks
Testing
  • Normal data
  • Boundary data
  • Invalid / erroneous data
Types of error
  • Syntax error: breaks language rules.
  • Logic error: program runs but gives the wrong result.
  • Runtime error: happens while the program is running.

7. Translators and IDEs

Translators
  • Compiler: translates whole program before execution.
  • Interpreter: translates and executes line by line.
  • Assembler: translates assembly language into machine code.
IDE features
  • Code editor
  • Syntax highlighting
  • Auto-complete
  • Error diagnostics
  • Debugger
  • Run tools
